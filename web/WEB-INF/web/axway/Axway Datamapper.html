Datamapper is a set of development tools, used for designing and building conversion programs. The conversion programs are then used in integrations and executed on the XIB Server. A conversion program translates between different file formats, but its operation can also include program access to various applications.

With Datamapper you can design conversion programs that can:

  • Read standard EDI documents
  • Write standard EDI documents
  • Read XML documents
  • Write XML documents
  • Read application specific documents (in-house files)
  • Write application specific documents
  • Access applications through their Application Programming Interface (hereafter abbreviated to API)

Datamapper Client is a program that runs on a PC It has no direct connection with the XIB server. Datamapper produces files that are used to generate components that executes on the XIB Server.

Locally on the PC, Datamapper Client:

  • Creates and revises files describing application documents or application interfaces (ADF files).
  • Creates and revises files describing the mapping.
  • Reads the EDI standard tables describing the EDI documents.
  • Creates files specifying the map (Map files).
  • Creates files describing how to perform the mapping (Run-Time Macro files, hereafter abbreviated to RTM).

The ADF(Application Data Format) files, RTM files, and the EDI standard tables are required for generating the components. The ADF files and RTM files are automatically copied to the server whenever they are revised by Datamapper Client. The component is built, tested with Datamapper Builder, and registered as a component to be run on the XIB Server.

Introduction

ADF is Application Data Format. The ADF file defines the document structure of your data. It is built with records and fields. The records group different fields together into logical groups. The fields contain the data.

An ADF can be of several types, for example EDI, inhouse, XML.

Create ADF

  1. File > New > Application[show]
  2. Fill the General:[show]
    • Title: the same as ADF name, please see the naming convertion.
    • Description: type the description
    • Directives: see the Directives tab
  3. Fill the Delimiters: [show]. This is the default setting, you can overwrite the delimiters when you define the Record/Field.
    • Record Delimiter: Fixed / Character / String / Implicit
      • Fixed: each Record has fixed length and system will read the Record by length.
      • Character: each Record ends with the specify character and system will trade the value after the character as a new Record.
      • String: each Record ends with the specify string and system will trade the value after the string as a new Record.
      • Implicit: system will read the Records depend on the field delimiter setting.
    • Field Delimiter:
      • Fixed: each Field has fixed length and system will read the Field by length.
      • Character: each Field ends with the specify chara cter and system will trade the value after the character as a new Field.
      • String: each Field ends with the specify string and system will trade the value after the string as a new Field.

Complete the ADF

There are two elements you can choose to create a ADF, but they have different types:

  • Record, there are two types:
    • Record, if you are familiar with FF/X12/EDIFACT, this stands for segment or XML tag.
    • Component, it is a child of Record. If you are familiar with FF/X12/EDIFACT, this stands for component or XML tag.
  • Field, there are three types:
    • Landmark, this is the identifier of the Record.
    • Normal, it is a normal field in Record.
    • Breakpoint, it is a normal field, but it will group the same Record depends on this field.

Let's begin how to set the Record properties:

  • Name: the record name.
  • Description: the record description.
  • Directives: see the Directives part.
  • Delimiter: Type/Value refer to the ADF properties above.
  • Record Occurrences: input the number you want, if max is 0, it means no limit.

Let's begin how to set the field properties:

  • Name: the record name.
  • Description: the record description.
  • Directives: see the Directives part.
  • Delimiter: Type/Value refer to the ADF properties above.
  • Record Occurrences: input the number you want, if max is 0, it means no limit.
